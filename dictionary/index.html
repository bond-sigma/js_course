Типы данных

Типы данных:

  • Числа 
    var number_variable  = 10;
    Передаются по значению при копировании переменных
  • Строки 
    var string_variable  = "я строка";
    Передаются по значению при копировании переменных
  • Булевые или логические 
    var boolean_variable = true;
    Могут иметь только два значения true или false.
  • null 
    var null_variable = null;
    Обозначает что переменная пуста.
  • Объекты 
    var object_variable  = {};
    При копировании в переменные передается ссылка на объект. Объект при этом в памяти не копируется.

Операторы. Приведение типов. Условия. Switch
  • Операторы 
    || - Логическое ИЛИ. Срабатывает и возращает true если левая или правая часть true
== - Оператора НЕстрогого равенства. При сравнении конвертирует переменные разных типов в число.
=== - Оператора СТРОГОГО равенства. При сравнении не ковертирует.
&& - Оператор И . Срабатывает и возращает true если левая И правая часть true выражения true
! - Оператор отрицания. Приводит переменную к булевому значению и меняет на обратную.
>, < - Операторы сравнения БОЛЬШЕ, МЕНЬШЕ. Строки сравниваются не по длине, а по кодам букв в таблице.
>=, <= - Операторы сравнения БОЛЬШЕ ИЛИ РАВНО, МЕНЬШЕ ИЛИ РАВНО.
  • Приведение типов. 
    Приведение к числу осуществляется приставкой знака + перед переменной. - +"string"
Приведение к булевому значению осуществляется приставкой !! к переменной - !!"some_string"
  • Условные переменные. Используются для написания логики программ и их ветвления. 
    if ( true ) {
  console.log( 'Оператор условия в действии' );
}

if ( "Some_string" ) {
   console.log('Some_string');
}
    В условных операторах любая непустая строка, число не 0 является true.
  • Switch - используется как замена if else . 
            switch (day) {
            case "Суббота":
                x = "Сегодня Суббота";
                break;

            case "Воскресенье":
                x = "Сегодня Воскресенье";
                break;

            default:
                x = "Сегодня будний день";
                break;
        }

      BOM, DOM

      BOM

      Это набор глобальных переменных дляполучения информации о браузере и текущем УРЛ
      • window - является главным объектом браузера. Содежит в себе все глобавльные переменные. Является мостом между JS и DOM 
        console.log(window);
      • window.screen или screen - Содержит в себе информацию о экране пользователя. 
        console.log('Текущий размер экрана: Высота: ' + window.screen.height + " Ширина: " + window.screen.width);
      • navigator - Содержит в себе информацию о браузере пользователя. 
        console.log('Текущая информация о браузере: ' + navigator.userAgent);
console.log('ТВаша платформа: ' +  window.navigator.platform);
      • location - Содержит в себе всю информацию о текущем УРЛ страницы. Имеет методы для перехода на другую страницу или перезагрузки текущей. 
        console.log(location.href);
console.log('Текущий хост: ' + location.host);
console.log('Текущий путь: ' +  window.location.pathname);
console.log('Текущий протокол: ' +  window.location.protocol);
window.location.reload(); - метод для перезагрузки текущей страницы.

      DOM

      Объектная модель браузера отвечающая за управление HTML. Каждый HTML тег имеет представление в ввиде объекта.
      • getElementById - Получение ссылки на объект тега через его аттрибут id 
        var idsElement = document.getElementById('content-id');
      • getElementsByClassName - Поиск елементов по их аттрибуту class 
        var classElements = document.getElementsByClassName('test-class');
        Возращает объект HtmlCollection. По поведению схожий с массивом, но также имеет метод index для доступа к елементам по индексу в коллекции
      • getElementsByTagName - Поиск елементов по их аттрибуту class 
            var tagElements = document.getElementsByTagName('p');
        Возращает объект HtmlCollection. По поведению схожий с массивом, но также имеет метод index для доступа к елементам по индексу в коллекции

      Свойство DOM объектов

      • childNodes - содержит в себе массив всех дочерних елементов на один уровень ниже. Включает в себя теги и текстовые узлы.
      • children - содержит в себе массив всех дочерних елементов на один уровень ниже. Включает в себя только теги.
      • firstChild - Содержит в себе ссылку на первый дочерний елемент уровнем ниже
      • lastChild - Содержит в себе ссылку на последний дочерний елемент уровнем ниже
      • parentNode - Содержит в себе ссылку на родительский елемент уровнем выше
      • nodeType - Содержит в себе ТИП узла. Числовое значение. "1" : "ELEMENT_NODE", "2" : "ATTRIBUTE_NODE", "3" : 'TEXT_NODE', "4" : "COMMENT_NODE"
      • tagName - Свойство доступное только у узлов отвечающих за теги
      • innerHTML - Свойство содержащее весь HTML который находится внутри тега в ввиде текста. Свойство доступное только у узлов отвечающих за теги
      • outerHTML - Свойство содержащее весь HTML который находится внутри тега в ввиде текста и включает в себя и текст открывающий и закрывающий текущий тег. Свойство доступное только у узлов отвечающих за теги
      • attributes - Свойство содержит в себе массив всех аттрибутов узла.
      • nextSibling - Содержит в себе ссылку на следующий узел в дереве на том же уровне что и текущий.
      • previousSibling - Содержит в себе ссылку на предыдущий узел в дереве на том же уровне что и текущий.

      Создание, вставка, удаление новых элементов в объектном подходе

      • document.createElement - метод для создание нового объекта который отвечает за тег. Параметром принимает имя тега. После создания объекту можно выставлять все свойства и вызывать методы так же само как у элемента которые уже есть на странице. Созданный объект еще не вставлен в ДОМ дерево. 
        var childElement = document.createElement( 'p' );
childElement.id = 'new-p';
childElement.className = 'new-p-classname';
childElement.style.color = 'green';
childElement.innerHTML = " I'm new p tag";
      • element.appendChild - метод для добавления элемента внутрь другого элемента . Элемент вставляется вконец. Этот метод вызывается у елемента внутть которого вставляют. 
        parentElement.appendChild( childElement );
      • parentElement.insertBefore - метод для вставки элемента внутрь другого элемента в определенную позицию. Принимает 2 параметра - 1 параметр элемент который вставляют, второй - позицию внутри относительно которой вставляют. 
        var parentElement = document.getElementById( 'div-element' );
var childElement = document.createElement( 'a' );
childElement.id = 'new-p2';
childElement.className = 'new-p-classname2';
childElement.style.color = 'red';
childElement.innerHTML = " I'm new p tag";
parentElement.insertBefore( childElement, parentElement.children[0] );

      Работа с аттрибутами

      • Перечисление аттрибутов производится через цикл 
                    for ( var i = 0; i < element.attributes.length; i++ ) {
                if ( typeof element.attributes[i] != 'function' ) {
                    console.log( 'Имя: ' + element.attributes[i].nodeName + " Значение: ", element.getAttribute( element.attributes[i].nodeName ) );
                }
            }
      • nodeName - имя аттрибута в списке аттрибутов HTML елемента 
        element.attributes[i].nodeName
      • nodeValue - значение аттрибута 
        element.attributes[i].nodeValue
      • getAttribute - метод для получения значения аттрибута напрямую из элемента. 
        element.getAttribute( "ИМЯ АТТРИБУТА" );
      • setAttribute - метод для установки значения свойства атррибута 
        element.setAttribute( 'ИМЯ-АТТРИБУТА', 'НОВОЕ ЗНАЧЕНИЕ' );
      • removeAttribute - метод для удаления аттрибута из элемента 
        element.removeAttribute( 'ИМЯ-АТТРИБУТА' );
      • setNamedItem - добавление нового аттрибута. Добавление происходит через создание нового объекта который отвечает за новый аттрибут. Создание нового аттрибута происходит с помощью метода document.createAttribute и передачей туда параметра с именем нового аттрибута. 
        var ourNewAttribute = document.createAttribute( "custom" );
ourNewAttribute.nodeValue = "custom-value-attributes";
element.attributes.setNamedItem(ourNewAttribute);
      • insertAdjacentHTML - вставка нового HTML элемента в ввиде строки в ДОМ дерево в определенное место. insertAdjacentElement - вставка нового HTML элемента в ввиде объекта в ДОМ дерево в определенное место. insertAdjacentText - вставка нового текстового узла в ДОМ дерево в определенное место Первым параметром эти методы принимают место вставки и может быть одним из нескольких значений
        -- beforeBegin - означает место перед открывающим тегом
        -- afterBegin - сразу после открывающим тегом
        -- beforeEnd - перед закрывающим тегом
        -- afterEnd - после закрывающего тега 
        //Пример использования вставки HTML в ввиде строки
var testDiv = document.getElementById('test-id');
//Получаем ссылку на второй тег Р чтобы использовать его как точку для вставки
var pObj    = testDiv.getElementsByTagName('p')[1];
pObj.insertAdjacentHTML('beforeBegin', 'New link inserted by insertAdjacentHTML');

//Пример вставки через в HTML в ввиде объекта
var testDiv = document.getElementById('test-id');
//Получаем ссылку на первый тег Р чтобы использовать его как точку для вставки
var pObj    = testDiv.getElementsByTagName('p')[0];
var newLink = document.createElement('a');
newLink.style.color = 'green';
newLink.setAttribute('href', 'http://google.com');
//Задаем внутренний текст нового елемента
newLink.innerHTML = 'I new link with insertAdjasmentElement';
pObj.insertAdjacentElement('beforeEnd', newLink);
      • documentFragment - особый вид элемента. Создается как контейнер для вставки других элементов , но при вставке в ДОМ дерево в вставляется только содержимое фрагмента, а сама фрагмент исчезает. 
        for (var i = 0; i < 10; i++) {
   newElHolder = document.createElement('li');
   newElHolder.innerHTML = "I'm new div with index: " + i;
   fragment.appendChild(newElHolder);
}

//Вставляем наш фрагмент внутрь елемента. Вставляются только теги LI, а сама оболочка в которой они находились до вставки - пропадает.
testDiv.appendChild(fragment);
      • Важно: Любое изменение свойства innerHTML вызывает перерисовку ДОМ дерева, поэтому любое изменение должно осуществлятся как можно реже, если нужно что то вставлять в страницу в цикле , нужно собрать в цикле строку или сложить в один елемент и только после этого вставлять новое в ДОМ дерево. В таких случаях для сборки нового куска HTML возможно использовать documentFragment.
